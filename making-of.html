Estrutura

Árvore de arquivos

O site possui um padrão próprio de disposição de arquivos e diretórios, que é automaticamente processado com Webpack. Veja um resumo desse padrão abaixo.

  • raiar/ Submódulo com o framework web utilizado.
  • src/
    • assets/ Assets utilizados no site.
    • pages/
      • …/ Uma página web.
        • entry.ts Script exclusivo da página.
        • template.html HTML inicial da página.
        • … Demais arquivos de uso exclusivo da página.
    • scripts/
      • global/ Scripts executados em todas páginas.
      • … Scripts utilizados por outros scripts.
    • styles/ Estilos Sass utilizados no site.

Importação de módulos

Graças ao Webpack, vários diretórios podem ser referenciados via os seguintes aliases.

  • Scripts: Diretório contendo os scripts gerais.
  • Styles: Diretório contendo os estilos Sass.
  • RaiarComponents: Diretório de componentes oferecidos pelo framework Raiar.
Também é possível importar arquivos Sass das seguintes formas, utilizando resource queries.
  • ?string: Importa o Sass como uma string.
  • ?apply: Aplica o estilo Sass ao documento atual.
  • ?sheet: Carrega o estilo Sass como um objeto CSSStyleSheet.

Lista de projetos

A lista dinâmica de projetos obtém os projetos a partir de um módulo TypeScript contendo dados como nome, descrição, detalhes, tags, licença, URL, repositório etc. de cada projeto.

Os projetos são filtrados utilizando a classe ProjectQueryBuilder, que permite a especificação dos parâmetros de filtragem antes de obter os resultados do tipo ProjectQueryResult. Cada projeto armazena sua representação como elemento HTML, permitindo sua exibição. Aqui está um exemplo de filtragem.

const filteredProjects = new ProjectQueryBuilder(projects)
  .withTags(...requiredTags)
  .withLogic(filterMode)
  .get();

Raiar

O site é baseado em Raiar—um de meus projetos e um pequeno framework web fornecendo componentes JavaScript e estilos Sass configuráveis.
